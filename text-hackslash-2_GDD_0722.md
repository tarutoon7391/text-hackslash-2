## 0. 開発方針

- 旧コード・旧フォルダ・旧Railwayサービスは引き継がない。自宅PCの新規独立フォルダで1から実装する
- 旧リポジトリは `tarutoon7391/text-hackslash-2-old` に改名して保存。実装の参考として参照は可
- 新しいRailwayサービスを新規作成する
- 本GDDの「確定構造」は実装中に変更しない。「実装後調整」タグの付いた数値はプレイしながら調整する
- 数値は必ずデータファイルに分離し、ハードコーディング禁止（バランス調整をコード変更なしで行うため）

## 1. プロジェクト概要

- 初代text-hackslash（PHP+MySQL、テキストハクスラRPG）の後継作
- ストーリーは重視しない。システム・演出・周回体験の完成度を追求する
- ジャンル：ターン制テキストハックアンドスラッシュ、周回・収集・厳選型

## 2. 技術スタック

| 項目 | 内容 |
|---|---|
| バックエンド | Node.js + Express + Socket.io |
| DB | PostgreSQL on Railway（新規サービス） |
| フロントエンド | React + Vite |
| リポジトリ | 新規・独立フォルダ（旧BiblioNaviフォルダとは完全分離） |

## 3. UI要件（モバイルファースト）

- スマホ縦画面での快適プレイを最優先に設計。PCでも崩れないがスマホが基準
- タップ操作前提：ボタンは指で押しやすいサイズ（44px以上目安）、ホバー依存UI禁止
- スクロール最小主義：無駄なスクロールは全排除。スクロールが許されるのは本質的に一覧性が必要な部分（インベントリ・図鑑・戦闘ログ履歴等）のみ
- 戦闘・拠点メニュー等の主要画面は1画面完結が原則
- 抽出・移植UIは情報量が多いため、スマホでの操作動線を特に丁寧に設計する

## 4. 属性システム【確定構造】

**6属性**：火・水・雷・土・光・闇

```
火 → 土（1.5倍）
土 → 雷（1.5倍）
雷 → 水（1.5倍）
水 → 火（1.5倍）
光 ⇔ 闇（お互いに1.5倍）
```
- 弱点：×1.5／耐性：×0.5／通常：×1.0
- 武器の属性は「相性計算」と「属性一致ボーナス」にのみ使用。武器属性による状態異常の自動付与はなし

## 5. 弱体効果システム【確定構造】

**用語定義（この3語以外の表記は使用禁止）**
- **状態異常**：火傷・凍結・感電・呪い・混乱の5種
- **デバフ**：ステータス低下（ATK-X%等）
- **弱体効果**：状態異常＋デバフの総称

**状態異常5種**

| 状態異常 | 効果 | 対応属性 |
|---|---|---|
| 火傷 | 継続ダメージ＋回復量減少 | 火 |
| 凍結 | 行動不可、被弾で解除＋追加ダメージ | 水 |
| 感電 | 行動失敗率アップ | 雷 |
| 混乱 | 味方誤爆あり | 土 |
| 呪い | 回復無効・デバフ蓄積 | 闇 |

※光は状態異常を持たず、解除・耐性サポート枠

**管理ルール**
- 状態異常は「スタック数（効果量）×残りターン（持続）」の2軸管理
- バフ/デバフ：同効果の重ね掛け不可。同効果を再使用した場合、残りターン数を最後に使ったスキルの値で上書き
- 参照系スキル（「弱体効果があるなら+X倍」等）は状態異常とデバフの両方をカウント
- 解除系スキルは状態異常とデバフの両方から選択して解除可能
  - 単体解除：手動選択
  - 全体解除：各キャラの残りターン最長の弱体効果を自動選択（同数なら状態異常優先）
- 操作系スキル（強化）：状態異常はスタック+2、デバフは残りターン+2。対象は手動選択

**将来拡張メモ**：解除不可の弱体効果（ボス戦ギミック用）。データ構造に dispellable フラグを最初から持たせておくこと

## 9. スキルシステム【確定構造】

### 9.1 スキルプール
- **無属性スキル168個**（6武器種×28個：★1×8、★2×8、★3×6、★4×6）。詳細は付録スキル表参照
- **属性スキル48個**（6属性×8個：各星2種）。詳細は付録スキル表参照
- 状態異常の狙い撃ち付与・属性参照は属性スキルのみが持つ。無属性スキルはランダム付与・参照・解除・操作系は保持可

### 9.2 武器ドロップ時のスキル抽選

| レア度 | 枠数 | ★1 | ★2 | ★3 | ★4 | 属性スキル |
|---|---|---|---|---|---|---|
| コモン | 1 | 100% | ー | ー | ー | 付かない |
| レア | 2 | 80% | 20% | ー | ー | 無属性7:属性3 |
| エピック | 3 | 60% | 30% | 10% | ー | 無属性7:属性3 |
| レジェンダリー | 4 | 25% | 50% | 15% | 10% | 無属性7:属性3 |
| ユニーク | 固定 | 専用スキル固定 | | | | |

- 属性スキルは武器の属性と一致するもののみ出現
- 各枠50%で充填。全枠空振り時は1枠目に1個保証（全武器最低1スキル）
- この出現率はダンジョン深度で変化しない

### 9.3 属性一致ボーナス
- 武器属性とスキル属性が一致：威力+20%＋状態異常付与率+10%（数値は実装後調整）

### 9.4 スキルレベル（+値）
- 「スキル名(+N)」表記、上限なし
- ダンジョン（階層ブロック）ごとに固定+値。そこでのドロップは必ずその+値が付く（抽選なし）
- 効果：攻撃系=威力↑・MP↓／回復系=回復量↑・MP↓／補助系=追加効果が段階的に付与
- 階層⇔+値の対応表、+1あたりの効果量：**実装後調整**

## 10. エンチャントシステム【確定構造】

### 10.1 一覧（12種）

**青天井組（7種）**：ATK+%／スキル威力+%／会心ダメージ+%／HP+%／MP+%／DEF+%／SPD+%
- 深層ほど際限なく伸びる。最終的に+1000%級も登場するインフレ設計

**渋カーブ組（5種）**：会心率+%／状態異常付与率+%／回避+%／被ダメージ軽減%／弱体効果耐性+%
- 100%上限のある有限ステータス。成長カーブを極端に渋くする

### 10.2 付与ルール
- 枠数：コモン0／レア1／エピック2／レジェンダリー3／ユニーク=固有効果+1〜2
- 各枠50%で充填、保証なし（エンチャント0個の装備もあり得る）
- 数値はドロップしたダンジョンの深度で決まるレンジからランダム抽選（対応表は**実装後調整**）
- 自然ドロップでは同種重複なし

### 10.3 重複ルール
- 渋カーブ組：1装備につき同種1個まで（移植でも重複不可）
- 青天井組：移植でのみ同種を複数積める。効果は単純加算

### 10.4 部位別の出現重み
- 武器：攻撃系×2.0／生存系×0.5／特殊系×1.0
- 防具：生存系×2.0／攻撃系×0.3／特殊系×1.0
- アクセサリー：全×1.0
- カテゴリ：攻撃系=ATK+/スキル威力+/会心ダメ+/会心率+/状態異常付与率+、生存系=HP+/DEF+/被ダメ軽減/弱体効果耐性+、特殊系=MP+/SPD+/回避+

## 11. 抽出・移植システム【確定構造】

- **抽出**：装備からスキルまたはエンチャントを1つだけ選んで取り出し、スキル/エンチャントインベントリに入れる。**元の装備は破壊される**
- **移植**：ステータス画面で装備のスキル/エンチャント枠を選択→「変更」でインベントリのものを**上書き**（元々付いていたものは消滅）
- 枠数はレア度依存のまま増えない
- 移植時も重複ルール（10.3）に従う

## 12. 装備システム【確定構造】

- スロット：武器1／防具1／アクセサリー2
- 装備の3軸：武器種（武器のみ）×属性×レア度
- レア度：コモン／レア／エピック／レジェンダリー／ユニーク
- ダンジョンの深さに応じて同レア度でも基礎ステータス上限が上昇
- ドロップ傾向：雑魚=コモン〜レア中心、低確率エピック／エリート=レア〜エピック中心、低確率レジェンダリー／ボス=エピック〜レジェンダリー確定、極低確率ユニーク（率は**実装後調整**）
- 消耗品・素材も強力なものを用意（内容は**実装後調整**）

## 13. モンスター仲間システム【確定構造】

- **捕獲（ドラクエ5方式）**：敵撃破時に種族ごとのcaptureRateで抽選→成功で仲間化確認ダイアログ
- 捕獲率目安：雑魚10%／エリート1%／ボス0%（実数値は実装後調整）
- パーティ：プレイヤー1人＋モンスター最大3体＝4体編成。拠点で同行3体を選択
- 成長：経験値レベルアップ、種族ごとの成長曲線
- **装備：モンスターは武器種制限なし、全装備を自由に装備可能（暫定。バランス問題が出たら種族別制限を検討。equipableWeaponTypesフィールドを種族データに用意しておく）**
- 種族固有スキル保持＋装備武器のスキルも使用可能
- 連携はA系統（弱体効果コンボ）のみ対象、B系統（職業連携）は対象外
- 種族リストは実装時にゾーンテーマに合わせて量産

## 14. ダンジョン構造【確定構造】

- **エンドレスタワー方式**＋**チェックポイント制**：5階ごと（1・6・11…）にチェックポイント。ブロックのボス撃破で次のチェックポイント解放。拠点から到達済みチェックポイントを選んで出発
- **属性ゾーンテーマ**：5階ブロックごとに6属性を巡回（1〜5階=火、6〜10階=水…30階で一周）。ドロップの属性傾向がゾーンに連動
- **敗北・撤退時**：常にロビーに戻る。ロビーで再出発地点を選ぶ
- 序盤（1〜5階）：初期装備・レベルで雑魚を確実に倒せる難易度、単体エンカウントのみ
- 深度に応じて上昇するもの：敵の強さ／装備基礎ステータス上限／スキル+値（固定値）／エンチャント数値レンジ
- 各対応表は**実装後調整**

## 15. 連携技システム【確定構造】

- **連携技A（弱体効果コンボ）**：属性スキルの追撃スキル群として実装（例：凍結中に「砕氷」等）。ソロ・協力・モンスター全対応
- **連携技B（パーティ職業連携）**：同ターンに特定職業の組み合わせでスキル使用→自動発動。人間キャラのみ。詳細は協力プレイ実装時に設計
